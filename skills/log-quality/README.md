# 日志质量链路

覆盖日志质量检查、整改、新写全生命周期，依赖外部标准文档 `log-quality-standard` 作为规范基准。

## 包含技能

| 技能 | 说明 | 触发词 |
|------|------|--------|
| log-quality-standard | 日志标准规范适配器，定位外部标准文档 | 日志标准、日志规范、log quality |
| log-quality-scan | 扫描日志文件，逐条判断是否符合标准 | 扫描日志、日志质量检查 |
| log-quality-scan-code | 从代码仓库扫描日志，检查质量并检测遗漏 | 代码仓日志质量检查 |
| log-quality-rewrite | 将不合规日志改写为符合标准的版本 | 重写日志、日志整改 |
| log-quality-write | 从零为新功能编写符合标准的日志方案 | 写日志、编写日志、日志设计 |

## 生命周期

```
新功能设计  →  log-quality-write（先写）
        ↓
代码实现后  →  log-quality-scan-code（查质量 + 查遗漏）
        ↓
发现问题    →  log-quality-scan（逐条确认）
        ↓
整改        →  log-quality-rewrite（批量改写）
```

## 使用方式

- 制定/查看日志规范：说 `日志标准`
- 检查现有日志质量：说 `扫描日志` 或 `代码仓日志质量检查`
- 整改不合规日志：说 `重写日志`（需先有 scan 报告）
- 为新功能设计日志：说 `写日志`
