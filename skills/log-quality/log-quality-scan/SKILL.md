# 日志质量扫描器

对日志文件或代码中的日志语句进行质量检查，判断每条日志是否符合 `log-quality-standard` 的 6 条标准。

---

## 触发词

扫描日志、判断日志好坏、日志质量检查、日志扫描

---

## 输入

| 输入 | 必需 | 说明 |
|------|------|------|
| 日志内容 | 必需 | 日志文件路径（.log/.txt）或直接粘贴的日志内容 |
| 代码路径 | 可选 | 如果是检查代码中的日志语句，提供代码根目录 |

---

## 输出

| 输出 | 格式 | 说明 |
|------|------|------|
| 扫描报告 | Markdown | 每条日志的检查结果 + 不合规原因 + 修改建议 |
| 汇总统计 | 数字 | 总数、通过数、不通过数、各标准覆盖率 |

---

## 检查标准（对应 log-quality-standard）

| 标准 | 检查项 |
|------|--------|
| 标准1：分级清晰 | 异常日志是否为 ERROR/WARNING？INFO 日志是否确实属于正常流程？ |
| 标准2：描述充分 | ERROR/WARNING 是否回答了"什么错+为什么+查哪里"？参数错误是否打印了期望值和实际值？ |
| 标准3：组件归属 | 每条日志是否有明确的组件标识（仓库/子模块）？ |
| 标准4：防刷屏 | 高频重复日志是否合并计数？循环内是否逐条打印？ |
| 标准5：链路追踪 | ERROR/WARNING 是否带 trace_id/request_id？ |
| 标准6：隐私保护 | 是否包含用户原始输入、prompt、PII、完整 token？ |

---

## 工作流程

### 第一步：解析日志

如果输入是文件路径，读取文件内容。
如果输入是纯文本，按行分割，每行当作一条日志。

### 第二步：逐条检查

对每条日志应用 6 条标准：

```
日志内容：「[ERROR] link failed」
  ↓
标准1（分级）: ✓ 异常日志是 ERROR
标准2（描述）: ✗ 缺少组件标识、缺少根因、缺少分析方案
标准3（组件）: ✗ 无组件标识
标准4（防刷屏）: N/A（单条无法判断，需看上下文）
标准5（链路）: ✗ 无 trace_id
标准6（隐私）: ✓ 未发现 PII
```

### 第三步：汇总评级

| 评级 | 条件 |
|------|------|
| ✅ 合格 | 通过全部 6 条标准 |
| ⚠️ 警告 | 通过 ≥4 条标准，但有 1~2 条不通过 |
| ❌ 不合格 | 不足 4 条标准通过，或任何一条关键标准（标准2/3/6）不通过 |

### 第四步：输出报告

```markdown
# 日志质量扫描报告

## 汇总统计
- **总日志数**：X 条
- **通过**：X 条（Y%）
- **警告**：X 条
- **不合格**：X 条

## 各标准覆盖率
| 标准 | 通过率 |
|------|--------|
| 标准1（分级清晰） | 90% |
| 标准2（描述充分） | 60% |
| 标准3（组件归属） | 75% |
| 标准4（防刷屏） | 80% |
| 标准5（链路追踪） | 50% |
| 标准6（隐私保护） | 95% |

---

## 逐条检查结果

| # | 日志内容（截断） | 分级 | 描述 | 组件 | 防刷屏 | 链路 | 隐私 | 评级 | 不合规原因 | 改进建议 |
|---|-----------------|------|------|------|--------|------|------|------|-----------|----------|
| 1 | [ERROR] link failed | ✓ | ✗ | ✗ | N/A | ✗ | ✓ | ❌ | 缺少组件、根因、trace_id | 改为 `[ERROR] [cmotor/router] trace_id=xxx, Link failed, error=..., 可检查...` |
```

---

## 不合规判定规则

| 场景 | 判定 | 说明 |
|------|------|------|
| 异常日志打 INFO | ❌ 不合格 | 严重违规，分级不清 |
| ERROR 无组件标识 | ❌ 不合格 | 无法定位 |
| ERROR 无 trace_id | ⚠️ 警告 | 无链路追踪能力 |
| 含用户 prompt | ❌ 不合格 | 隐私泄露 |
| 含完整 api_key | ❌ 不合格 | 隐私泄露 |
| 参数错误无对比值 | ⚠️ 警告 | 描述不充分 |
| 循环内逐条打 ERROR | ⚠️ 警告 | 需合并计数 |

---

## 注意事项

- 如果输入是代码文件，先用 grep 提取所有日志语句，再逐条检查
- 日志内容中的占位符（`{id}`、`%s`）不纳入隐私检查
- 单条日志无法判断防刷屏时，标注"N/A"，以汇总统计中的重复模式分析为准
- 建议配合 log-quality-rewrite 使用，scan 完成后无缝进入 rewrite
